Code Log 1: HTML Semantic Tags

Non-seantic tags act as anchors or boxes where the content is placed. Often, these are used for layouts or sectioning off the code into boxes. For example, we often see these as the div or span tag for an area of code.

Semantic tags on the other hand are more specific for labeling and describing parts of the code from the header to the footer and beyond.

More specifically, there are a few key benefits of using semantic tags. For example:

  • Cleaner Code - Semantic tags specifically label what each section of the code does, helping not only you as the coder understand what is happening but also anyone else who reads the code after you.
  • Structured Code - While the labeling is helpful, the tags also help create a hierachy for a section that is easy to follow. Such as a header above a paragraph tag.
  • Accessibility - Most screen readers rely on the semantic tags to easier navigate a page for those with disabilities.
